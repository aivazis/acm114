Schedule of lectures

This page contains outlines of the lectures given during the Winter 2010 term. You can use the calendar on the left to navigate to a specific lecture where you will also find instructions on how to download the lecture notes in PDF form. If you are interested in the LaTeX source code, you will have to gain access to the bazaar source repository. Let me know and I will send you instructions.

Please note that there are no lectures for January 18 and February 15; they are Institute holidays. The last day of class is March 10, and final projects are due on March 15.

Monday, January 4

Introductions. The syllabus. Course scope and outline. Motivation and overview of high performance computing in science. Informal student survey: background, experience, personal computing platforms, access to computational resources.

Monday, January 11

Illustration of parallel programming models with a simple problem: a global reduction. Shared and distributed address spaces; threads and message passing. Example pitfalls: race conditions. Layout of a SIMD machine and data parallel programming. Clusters of SMPs and hybrid programming models. Bulk synchronous programming and algorithmic phases.

Wednesday, January 13

Performance metrics; speedup and efficiency. The bad news: Amdahl's law. Strong and weak scaling. Isoefficiency as a suitable scaling metric. Reducing the sequential computation fraction as the problem size grows by algorithmic improvements. Sources of parallel inefficiency. Modeling communication costs. Tensions in optimizing parallel efficiency. The good news: parallelism makes you a better programmer.

Login through ssh, configuring the shell, setting up the necessary packages -- compilers, mpi, other useful tools. Source controls systems: cvs and bazaar. The coding cycle: editing, compiling and linking, running, testing. Configuration management: make, config.

Monday, February 8

Structured grids. Overview, advantages and disadvantages. Data layout and single node performance for complex physics. Grid updates. Implementation strategy for the sequential, threaded and MPI codes. The scaffolding for the sequential solver.
